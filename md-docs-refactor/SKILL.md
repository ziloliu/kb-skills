# Markdown文档多级主题重构与合并技能

## 概述

此技能专门用于建立多级主题结构的Markdown文档重构。它深度分析多个.md文件的内容，提取主主题、子主题和子子主题，建立清晰的2-3级主题层级结构，然后将所有内容合并为一个以根主题为主的.md文档，内部子主题按照逻辑顺序排列。最终输出一个完整的、结构清晰的合并文档。

## 核心工作流程

### 1. 多级主题分析阶段
- 读取指定文件夹中的所有.md文件（通常不超过10个）
- 深度分析每个文件的内容，识别潜在的主主题、子主题和子子主题
- 基于关键词、标题层级和内容语义建立清晰的2-3级主题结构
- **必须建立明确的层级关系**：主主题 → 子主题 → 子子主题
- **示例结构**：
  ```
  技术文档（主主题）
  ├── 系统架构（子主题）
  │   ├── 整体设计（子子主题）
  │   └── 技术栈（子子主题）
  ├── 开发指南（子主题）
  │   ├── 环境搭建（子子主题）
  │   └── 代码规范（子子主题）
  └── 部署流程（子主题）
      ├── 部署环境（子子主题）
      └── 监控（子子主题）
  ```
- 特别关注用户需求中的"建立多级的主题结构"要求

### 2. 重构阶段
1. **建立主题框架**：基于分析结果创建清晰的主题层级结构
2. **内容映射**：将每个文档中的内容点映射到对应的主题下
3. **内容填充**：将内容按主题组织，保持原文措辞和原意
4. **格式保留**：确保代码块、表格、链接、图片等特殊格式完整保留
5. **清理冗余**：删除无法映射到任何主题的内容

### 3. 主题排序与合并阶段
1. **根主题确定**：识别并确定一个核心根主题作为整个文档的顶层主题
   - **重要边界情况**：如果无法确定一个共同的根主题（多个独立的主主题，它们之间没有明显的层级关系），则不要强制合并
   - 在这种情况下，分别为每个主主题生成独立的.md文件
2. **子主题逻辑排序**：
   - 按照概念的逻辑顺序排列子主题（从基础到高级、从概述到细节、从原因到结果等）
   - 考虑主题之间的依赖关系，确保前置概念先出现
   - 保持主题的连贯性和可读性
3. **文档生成**：
   - **有共同根主题**：将所有内容整合到一个以根主题为主的.md文件中
   - **无共同根主题**：为每个主主题生成独立的.md文件，每个文件内部按子主题层级组织
4. **输出**：保存到指定目录

## 子主题逻辑排序原则

**必须遵循以下排序原则：**

1. **概念递进原则**：从基础概念到高级概念，从简单到复杂
2. **依赖关系原则**：被依赖的主题先出现，依赖其他主题的后出现
3. **问题-解决原则**：先描述问题，再提供解决方案
4. **时间顺序原则**：按事件发生的先后顺序排列
5. **空间顺序原则**：按物理或逻辑空间顺序排列
6. **重要性顺序原则**：最重要的主题先出现，次要的后出现

**排序示例：**
```
根主题：项目开发指南
├── 项目概述（基础概念）
├── 环境搭建（依赖于概述）
├── 基础功能开发（依赖于环境）
├── 高级功能开发（依赖于基础功能）
├── 测试与部署（依赖于功能开发）
└── 常见问题与解决方案（最后）
```

## 详细指令

### 输入要求
- 用户提供一个包含多个.md文件的文件夹路径
- 文件数量通常不超过10个
- 文件内容应为中文或英文Markdown格式

### 处理步骤

#### 步骤1：读取和分析
1. 使用`glob`工具查找指定文件夹中的所有.md文件
2. 使用`read`工具读取每个文件内容
3. 分析每个文件，识别：
   - 标题层级（#、##、###）
   - 关键词和关键概念
   - 内容段落和列表
   - 代码块、表格等特殊元素

#### 步骤2：主题提取
1. 从所有文档中提取共同的主题
2. 建立主题层级关系：
   - 主主题（顶层概念）
   - 子主题（支持性概念）
   - 子子主题（具体细节）
3. 主题应基于内容语义，而非单纯的字面匹配

#### 步骤3：内容映射
1. 将每个文档的段落、列表项映射到最相关的主题
2. 使用语义相似性判断内容与主题的匹配度
3. 如果内容无法映射到任何主题，考虑：
   - 是否创建新的主题
   - 是否删除无关内容（如果与整体主题无关）

#### 步骤4：主题排序与文档生成
1. **确定根主题**：
   - 从所有主题中选择最核心、最概括的主题作为根主题
   - 根主题应该能够涵盖所有其他主题的内容
   - **边界检查**：判断是否存在一个明确的共同根主题
     - 如果有明确的共同根主题 → 执行单文档合并流程
     - 如果没有明确的共同根主题（多个独立主主题）→ 执行多文档生成流程
2. **子主题逻辑排序**：
   - 按照"子主题逻辑排序原则"对所有子主题进行排序
   - 确保排序后的主题流逻辑清晰、易于理解
   - 可以向用户展示排序方案供确认
3. **生成文档**：
   - **场景A：有共同根主题（单文档）**
     - 创建单个.md文件
     - 文件命名：使用根主题名称（kebab-case格式）
     - 文件结构：
       ```
       # 根主题名称
       
       ## 子主题1（按逻辑顺序排列的第一个）
       相关的内容段落...
       
       ## 子主题2（按逻辑顺序排列的第二个）
       相关的内容段落...
       
       ### 子子主题2.1
       更详细的内容...
       
       ## 子主题3（按逻辑顺序排列的第三个）
       ...
       ```
   
   - **场景B：无共同根主题（多文档）**
     - 为每个独立的主主题创建单独的.md文件
     - 文件命名：使用各主主题名称（kebab-case格式）
     - 每个文件内部按子主题层级组织
     - 文件结构（每个文件类似）：
       ```
       # 主主题A名称
       
       ## 子主题A1
       相关的内容段落...
       
       ## 子主题A2
       相关的内容段落...
       
       ### 子子主题A2.1
       更详细的内容...
       ```
4. 保持原文内容，只进行重组和排序，不重写

#### 步骤5：格式处理
1. 代码块：保持原样，包括语言标识
2. 表格：保持原样，包括对齐方式
3. 链接：保持有效，如果链接目标在重构后移动，需要更新路径
4. 图片：保持引用，确保图片文件路径正确

### 输出格式
**场景A - 有共同根主题：**
- 输出到`refactored-docs/`目录（或用户指定目录）
- 单个合并的.md文件，以根主题命名
- 文件内部按逻辑顺序排列的多级主题结构组织内容
- 根主题为一级标题（#），子主题为二级标题（##），子子主题为三级标题（###）

**场景B - 无共同根主题：**
- 输出到`refactored-docs/`目录（或用户指定目录）
- 多个独立的.md文件，每个文件以对应的主主题命名
- 每个文件内部按子主题层级组织内容
- 各主主题为一级标题（#），子主题为二级标题（##），子子主题为三级标题（###）

## 注意事项

1. **内容保留**：尽量保持原文措辞，只改变组织结构
2. **语义理解**：基于内容含义而非字面匹配进行主题分类
3. **层级深度**：主题结构不宜过深，通常2-3级为宜
4. **文件数量**：如果主题过多，考虑合并相关主题
5. **用户确认**：在最终输出前，可以向用户展示主题结构供确认

## 质量检查

完成重构后，检查：
1. 所有重要内容是否都保留
2. 主题结构是否逻辑清晰
3. 特殊格式（代码、表格）是否正确保留
4. 文件之间的引用是否仍然有效

## 常见用例

- 整理杂乱的项目文档
- 重构知识库，改善信息架构
- 合并多个来源的文档到统一结构
- 为现有文档创建更好的导航结构

## 技能触发说明

**必须使用此技能的情况**：
1. 当用户明确提到"建立多级主题结构"、"提炼主题和子主题"、"按主题层级组织内容"时
2. 当用户需要整理多个.md文档并建立清晰的2-3级主题层级时
3. 当用户提到"重构思路：首先提炼多个文档中的主题，以及每个主题下所包含的子主题，以及子主题下的子主题"时
4. 当处理文件夹中的多个.md文件（通常不超过10个）需要按主题重新组织时

**与kb-cook技能的区别**：
- **md-docs-refactor**：专注于建立多级主题层级结构，深度分析内容语义，建立主主题→子主题→子子主题的清晰层级
- **kb-cook**：专注于保留原文100%内容并优化格式，不强调多级主题结构的建立

**主动触发**：当用户提到任何与"多级主题"、"层级结构"、"主题和子主题"相关的文档整理需求时，立即使用此技能。
